## 📊 博弈交易法分析内容

分析结果包括：

1. **阶段分析**
   - 当前阶段（一阶段到五阶段）
   - 阶段描述

2. **洗盘分析**
   - 是否处于洗盘状态
   - 洗盘强度（0-1）

3. **情绪比例关系**
   - 判断结果（看涨/看跌/中性）
   - 详细说明

4. **O点分析**
   - O点价格
   - 相对O点涨幅

5. **交易建议**
   - 操作建议（买入/卖出/持有/观望）
   - 理由
   - 仓位建议
   - 操作策略

## 🔧 技术架构

### 工作流程

```
用户输入
  ↓
意图识别 (/api/intent/recognize)
  ↓
提取股票代码和意图类型
  ↓
调用分析接口 (/api/chat/stream)
  ↓
根据意图选择分析方式
  ├─ 博弈交易法分析
  │   ├─ 整体阶段分析
  │   ├─ O点识别
  │   ├─ 洗盘分析
  │   ├─ 出货分析
  │   ├─ 情绪比例关系
  │   └─ 多方/空方锚定
  └─ 常规分析
      ├─ 技术指标分析
      ├─ AI分析
      └─ 交易建议
  ↓
流式返回结果 (SSE)
  ↓
前端展示
```

## 📝 API接口说明

### 1. 意图识别接口

**POST** `/api/intent/recognize`

请求体：
```json
{
  "message": "分析000001的阶段和洗盘"
}
```

响应：
```json
{
  "intent": "game_theory",
  "stock_code": "000001",
  "confidence": 0.8,
  "keywords": ["阶段", "洗盘"]
}
```

### 2. 聊天流式接口

**GET** `/api/chat/stream`

参数：
- `stock_code`: 股票代码（必需）
- `analysis_type`: 分析类型（auto/regular/game_theory）
- `message`: 用户消息（可选，用于意图识别）
- `days`: 分析天数（默认180）
- `run_backtest`: 是否回测（默认true）
- 其他回测参数...

## ⚙️ 配置说明

可以在前端设置中配置：
- **分析天数**：默认180天
- **是否回测**：默认开启
- **回测参数**：均线窗口、成交量比率等

## 🎯 功能特点

- ✅ **智能意图识别**：自动识别用户需求
- ✅ **自动提取股票代码**：从消息中提取6位股票代码
- ✅ **流式返回**：实时显示分析进度
- ✅ **美观展示**：卡片式展示分析结果
- ✅ **完整分析**：包含阶段、洗盘、情绪、O点等
- ✅ **交易建议**：基于博弈交易法的操作建议

## 📌 注意事项

1. **股票代码**：必须是6位数字（A股代码）
2. **数据要求**：博弈分析需要至少60个交易日的数据
3. **网络**：如果网络不稳定，建议设置 `skip_realtime=true`
4. **意图识别**：如果无法识别，默认使用常规分析

## 🐛 故障排除

### 问题1：无法识别意图
- 检查是否包含博弈交易法关键词
- 确保股票代码格式正确（6位数字）

### 问题2：分析失败
- 检查股票代码是否正确
- 确认数据是否足够（至少60个交易日）
- 查看后端日志

### 问题3：前端无法连接
- 确认后端服务已启动（端口8000）
- 检查CORS设置
- 查看浏览器控制台错误
