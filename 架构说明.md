# 博弈交易法分析系统 - 架构说明

## 一、整体架构

基于LangGraph框架的多Agent系统，实现博弈交易法的完整分析流程。

### 核心设计理念

1. **模块化设计**：每个分析步骤都是独立的Agent，职责清晰
2. **状态驱动**：使用LangGraph的状态管理，在节点间传递分析结果
3. **条件分支**：支持常规分析和博弈分析两种路径
4. **可扩展性**：每个Agent可以独立优化和增强

## 三、工作流程详解

### 3.1 状态定义

`AnalysisState` 是工作流的核心状态，包含：
- 用户输入：`user_message`, `stock_code`, `analysis_type`
- 数据：`stock_data`
- 分析结果：各个Agent的分析结果
- 最终输出：`final_report`

### 3.2 节点流程

#### 1. 意图识别节点 (`intent_recognition`)
- 从用户消息中提取股票代码
- 识别分析类型（常规分析 vs 博弈分析）
- 基于关键词匹配

#### 2. 数据获取节点 (`data_fetch`)
- 使用akshare获取股票数据
- 数据预处理和清洗

#### 3. 路由节点 (`route`)
- 根据分析类型决定走哪个分支
- 博弈分析 → `stage_analysis`
- 常规分析 → `regular_analysis`

#### 4. 博弈分析链（顺序执行）

**阶段分析节点** (`stage_analysis`)
- 分析股票当前阶段（一至五阶段）
- 使用技术指标和规则判断
- 输出：阶段编号、阶段名称、描述、置信度

**O点识别节点** (`o_point_analysis`)
- 识别原始低点（O点）
- 计算相对O点涨幅
- 输出：O点价格、日期、相对涨幅

**洗盘分析节点** (`washout_analysis`)
- 识别洗盘类型（K线洗盘 vs 波段洗盘）
- 计算洗盘效果（恐惧程度、焦虑程度、场内散户数量）
- 输出：洗盘类型、强度、是否完成

**出货分析节点** (`distribution_analysis`)
- 识别出货规模（大规模/中等规模/小规模）
- 判断出货持续时间
- 输出：出货规模、操作建议

**情绪比例关系节点** (`emotion_ratio`)
- 分析出货形态的"好看程度"
- 分析洗盘形态的"难看程度"
- 判断后市方向（看涨/看跌/中性）
- 输出：方向、置信度、详细说明

**锚定分析节点** (`anchor_analysis`)
- 识别多方锚定形态
- 识别空方锚定形态
- 判断锚定是否形成/破坏
- 输出：锚定类型、状态、信号

**分析总结节点** (`summary`)
- 综合所有分析结果
- 生成分析总结报告
- 输出：总结文本

#### 5. 常规分析节点 (`regular_analysis`)
- 计算技术指标（MA、MACD、RSI等）
- 生成常规分析报告
- 输出：指标值、交易建议

#### 6. 策略推荐节点 (`strategy_recommendation`)
- 基于所有分析结果生成交易策略
- 根据阶段和情况给出操作建议
- 输出：操作建议、理由、仓位建议、止损建议

#### 7. 最终报告节点 (`final_report`)
- 生成最终分析报告
- 整合所有分析结果
- 输出：完整报告文本

#### 8. 回溯测试节点 (`backtest`) - 可选
- 根据策略进行历史回测
- 计算收益率、最大回撤等
- 输出：回测结果

## 四、Agent设计模式

每个Agent遵循以下模式：

```python
def agent_node(state: AnalysisState) -> AnalysisState:
    """
    1. 从state中获取输入数据
    2. 调用工具函数进行分析
    3. 使用LLM进行推理（可选）
    4. 更新state中的结果
    5. 返回更新后的state
    """
    # 获取数据
    stock_data = state.get('stock_data')
    
    # 调用工具
    result = analyze_tool(stock_data)
    
    # 更新状态
    state['result'] = result
    
    return state
```

## 五、工具模块设计

### 5.1 数据获取工具
- `fetch_stock_data()`: 获取股票K线数据
- `get_stock_name()`: 获取股票名称

### 5.2 技术分析工具
- `calculate_ma()`: 计算移动平均线
- `calculate_macd()`: 计算MACD指标
- `calculate_rsi()`: 计算RSI指标
- `find_o_point()`: 识别O点
- `identify_stage_indicators()`: 识别阶段指标
- `calculate_washout_indicators()`: 计算洗盘指标

### 5.3 形态识别工具
- `recognize_kline_pattern()`: 识别K线形态
- `identify_washout_type()`: 识别洗盘类型
- `identify_distribution_scale()`: 识别出货规模

### 5.4 计算工具
- `calculate_washout_effect()`: 计算洗盘效果
- `calculate_emotion_ratio()`: 计算情绪比例关系

## 六、扩展点

### 6.1 LLM集成
每个Agent都可以集成LLM进行更智能的分析：

```python
from langchain.llms import OpenAI

llm = OpenAI()

def analyze_with_llm(data, indicators):
    prompt = f"""
    基于以下数据和技术指标，分析股票阶段：
    数据：{data}
    指标：{indicators}
    """
    result = llm(prompt)
    return result
```

### 6.2 新增分析节点
1. 创建新的Agent文件
2. 实现agent_node函数
3. 在`analysis_graph.py`中添加节点和边

### 6.3 优化分析算法
- 阶段识别：可以使用机器学习模型
- 洗盘识别：可以优化洗盘公式
- 情绪比例：可以使用LLM判断形态美观度

## 七、API接口设计

### 7.1 流式接口
使用SSE（Server-Sent Events）实时返回分析进度：

```
GET /api/chat/stream?message=分析000001&stock_code=000001
```

返回格式：
```
data: {"type": "progress", "message": "分析股票阶段..."}
data: {"type": "result", "report": "..."}
data: {"type": "done"}
```

### 7.2 非流式接口
一次性返回完整结果：

```
POST /api/chat
{
  "message": "分析000001",
  "stock_code": "000001",
  "analysis_type": "auto"
}
```

## 九、错误处理

每个Agent都应该有错误处理：

```python
def agent_node(state: AnalysisState) -> AnalysisState:
    try:
        # 分析逻辑
        result = analyze()
        state['result'] = result
    except Exception as e:
        state['result'] = {'error': str(e)}
    return state
```
