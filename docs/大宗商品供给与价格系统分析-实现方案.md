# 大宗商品供给与价格系统分析 — 完整实现方案

## 一、目标与范围

### 1.1 业务目标

- **主目标**：实现大宗商品（含产业链）的**供给线路、价格、国际消息**等信息的**系统整理**，并基于大模型 Agent 做**结构化分析**与**自动判定、自动搜索补充**。
- **子目标**：  
  1. **数据整合**：从公开网站/接口获取并整理供给链、价格、国际消息，形成统一「原始证据包」。
  2. **智能分析**：调用大模型 Agent 对证据包做摘要、分类、驱动因子提取、置信度与矛盾识别。
  3. **闭环补充**：Agent 自动识别信息缺口与关键点，生成「建议搜索查询」，执行搜索后合并结果并可选多轮再分析，直至满足终止条件。

### 1.2 功能范围

| ------- | ------------------------------------ | --- |

### 1.3 非功能目标

- **可用性**：采集与搜索具备重试、超时、降级（部分源失败仍可产出报告）。  
- **合规**：遵守各站 robots.txt、限频、User-Agent 标识；不落库未脱敏个人数据。  
- **可维护性**：数据模型、Agent 输入输出、API 有清晰 schema；关键步骤可配置（如最大轮次、超时）。

### 1.4 边界与假设

- **数据源**：仅使用公开可访问的网站/API，不依赖商业授权数据库（若后续接入再扩展）。  
- **品种范围**：首期以常见大宗（如原油、铜、大豆、铁矿等）及 1～2 条产业链为主，品种与产业链列表可配置扩展。  
- **语言**：采集与搜索支持中英文；报告与结构化输出以中文为主，可配置。

## 四、大宗商品期货买卖策略模块

### 4.1 策略类型与适用场景

#### 4.1.1 趋势跟踪策略

**适用场景**：市场存在明确趋势（上涨或下跌），基本面驱动因素持续

**策略逻辑**：
- 基于技术指标（MA、MACD、RSI、布林带等）识别趋势方向
- 结合基本面分析（供给过剩/短缺、需求变化、政策影响）确认趋势
- 多重时间框架确认（日线、周线、月线）

**技术指标组合**：
- 移动平均线系统：短期MA上穿长期MA为买入信号，下穿为卖出信号
- MACD：金叉买入，死叉卖出，结合柱状图变化确认强度
- RSI：超买超卖辅助判断，RSI>70警惕回调，RSI<30关注反弹
- 布林带：价格突破上轨可能延续上涨，跌破下轨可能延续下跌
- 成交量：趋势确认的重要辅助指标

**入场条件**：
- 多个技术指标同时发出同向信号
- 基本面因素支持趋势方向
- 成交量放大确认

**出场条件**：
- 技术指标出现反转信号
- 基本面因素发生重大变化
- 达到预设止损或止盈位

#### 4.1.2 套利策略

**跨期套利**：
- 同一品种不同交割月份合约之间的价差交易
- 基于季节性规律、库存周期、供需预期
- 近月与远月价差偏离合理区间时建仓

**跨品种套利**：
- 相关品种之间的价差交易
- 原油与成品油（汽油、柴油）
- 大豆与豆粕、豆油
- 铁矿与螺纹钢
- 焦煤与焦炭

**跨市场套利**：
- 同一品种在不同交易所的价差交易
- 国内外价差（如原油WTI与布伦特）
- 考虑汇率、运输成本、关税等因素

**套利风险控制**：
- 价差偏离历史均值超过2倍标准差时建仓
- 设置止损位，防止价差继续扩大
- 关注基本面变化，避免结构性套利失败

#### 4.1.3 套期保值策略

**生产者套保**：
- 卖出期货合约锁定未来销售价格
- 适用于农产品、矿产等生产者
- 防止价格下跌风险

**消费者套保**：
- 买入期货合约锁定未来采购成本
- 适用于加工企业、贸易商
- 防止价格上涨风险

**套保比例**：
- 基于风险敞口确定套保比例
- 一般建议50%-80%的套保比例
- 根据市场波动性动态调整

#### 4.1.4 事件驱动策略

**政策事件**：
- OPEC+减产协议
- 美联储货币政策
- 贸易政策变化
- 环保政策调整

**天气事件**：
- 农作物生长期天气（干旱、洪涝、霜冻）
- 极端天气对能源需求影响
- 拉尼娜/厄尔尼诺现象

**地缘政治**：
- 战争冲突
- 制裁措施
- 供应链中断

**事件策略执行**：
- 提前布局预期事件
- 事件结果公布后快速反应
- 设置严格止损控制尾部风险

### 4.4 技术指标计算模块

**移动平均线系统**：
- SMA（简单移动平均）：SMA(n) = sum(close, n)/n
- EMA（指数移动平均）：EMA(n) = (close * 2/(n+1)) + (prev_ema * (1-2/(n+1)))
- 均线多头排列：短期>中期>长期
- 均线空头排列：短期<中期<长期

**MACD指标**：
- DIF = EMA(12) - EMA(26)
- DEA = EMA(DIF, 9)
- MACD柱 = (DIF - DEA) * 2
- 金叉：DIF上穿DEA，死叉：DIF下穿DEA

**RSI指标**：
- RSI(n) = 100 - 100/(1 + RS)
- RS = 平均涨幅/平均跌幅
- 超买区：RSI > 70，超卖区：RSI < 30

**布林带**：
- 中轨 = SMA(20)
- 上轨 = 中轨 + 2 * STD(20)
- 下轨 = 中轨 - 2 * STD(20)
- 带宽 = (上轨 - 下轨) / 中轨

**ATR（平均真实波幅）**：
- TR = max(high-low, |high-prev_close|, |low-prev_close|)
- ATR(n) = SMA(TR, n)

### 4.5 仓位管理与风险控制

**凯利公式仓位管理**：
- f* = (bp - q) / b
- f*：最优仓位比例
- b：盈亏比（平均盈利/平均亏损）
- p：胜率
- q：败率（1-p）

**固定比例仓位管理**：
- 单笔交易风险不超过总资金的2%
- 根据止损距离计算仓位：仓位 = 总资金 * 风险比例 / 止损距离

**波动率调整仓位**：
- 高波动品种降低仓位，低波动品种提高仓位
- 调整后仓位 = 基础仓位 * (基准波动率 / 当前波动率)

**止损策略**：
- 技术止损：基于关键技术位（支撑/阻力、均线、前低/前高）
- 时间止损：持仓时间超过预期未达到目标
- 资金止损：单笔亏损达到预设比例
- 波动率止损：基于ATR设置动态止损

**止盈策略**：
- 固定比例止盈：盈利达到预设比例平仓
- 移动止盈：价格有利时上移止损位
- 分批止盈：达到不同目标分批平仓
- 技术止盈：出现反转信号时平仓

### 4.6 策略回测与评估

**回测框架**：
- 使用历史数据模拟策略执行
- 考虑交易成本（手续费、滑点）
- 模拟真实市场环境（流动性、涨跌停）

**评估指标**：
- **收益率**：总收益率、年化收益率
- **风险指标**：最大回撤、波动率、VaR
- **风险调整收益**：夏普比率、索提诺比率、卡玛比率
- **交易统计**：胜率、盈亏比、平均持仓时间、交易频率

**回测注意事项**：
- 避免前视偏差（使用未来数据）
- 考虑交易成本和滑点
- 样本外验证（避免过拟合）
- 参数敏感性分析

### 4.7 策略Agent设计

**策略生成Agent**：
- 输入：市场分析结果、技术指标、新闻事件
- 输出：策略建议（StrategySignal）
- 职责：综合分析，生成具体交易建议

**风险评估Agent**：
- 输入：策略建议、历史数据
- 输出：风险指标、压力测试结果
- 职责：评估策略风险，提供风险控制建议

**回测Agent**：
- 输入：策略参数、历史数据
- 输出：回测结果（BacktestResult）
- 职责：执行历史回测，计算各项指标

**策略优化Agent**：
- 输入：回测结果、市场环境变化
- 输出：优化后的策略参数
- 职责：根据回测结果和市场变化优化策略

### 4.8 实时监控与调整

**策略监控指标**：
- 实时盈亏
- 持仓风险敞口
- 相关性变化
- 市场波动率
- 流动性状况

**触发调整条件**：
- 策略表现显著偏离预期
- 市场结构发生重大变化
- 相关性关系破裂
- 流动性大幅下降

**调整机制**：
- 动态调整仓位规模
- 修改止损止盈位
- 暂停或终止策略
- 切换到备用策略

    │  • 市场状态识别：趋势/震荡/转折、波动率水平、流动性状况                                   │
    └───────────────────────────────────────────┬───────────────────────────────────────────┘
                                                │
    ┌───────────────────────────────────────────▼───────────────────────────────────────────┐
    │  Layer 5：LLM 策略生成与优化（core/agents/strategy_agents）                             │
    │  • 综合基本面与技术面分析，识别交易机会                                                   │

    │  Layer 6：策略回测与风险评估（core/tools/backtest + core/agents）                      │
    │  • 历史回测验证策略有效性                                                               │
    │  • 计算风险指标：VaR、最大回撤、夏普比率、索提诺比率                                     │

    └───────────────────────────────────────────┬───────────────────────────────────────────┘
                                                │
    ┌───────────────────────────────────────────▼───────────────────────────────────────────┐
    │  输出：最终报告 + 结构化数据（供给链摘要、价格摘要、驱动因子、风险、建议、买卖策略）      │
    └───────────────────────────────────────────────────────────────────────────────────────┘

## 四、数据层设计

### 4.1 数据来源（公开网站/接口，建议优先）


| 类别     | 内容说明          | 可选公开来源（示例）                      |
| ------ | ------------- | ------------------------------- |
| 供给/产业链 | 产地、产能、库存、贸易流  | 行业协会、统计局、USDA、各交易所公告、研报摘要（可爬摘要） |
| 价格     | 现货、期货、指数、价差   | 交易所官网、东方财富/同花顺、akshare、新浪财经等    |
| 国际消息   | 政策、天气、地缘、供需新闻 | 新华社、Reuters/AP 摘要、交易所新闻、行业门户    |


说明：具体域名与接口在实现时按合规与可访问性选定，并做限频与缓存。

**数据源优先级与熔断**：  

- 价格类优先使用 akshare/交易所 API（稳定、有结构）；供给与新闻可配置多源顺序，按序尝试直至成功或全部失败。  
- 单源连续失败 N 次（如 3）可临时熔断该源一段时间（如 30 分钟），避免无效请求；熔断状态可写内存或 Redis，重启后重置。

### 4.2 统一数据结构（与 state 对齐）

- **供给线路**：产地/地区、品种、产能或产量、库存、贸易流向、主要企业/港口、数据来源、时间。
- **价格**：品种、合约/市场、价格类型（现货/期货/指数）、价格值、单位、时间、来源。
- **国际消息**：标题、摘要、来源、发布时间、关联品种/地区、标签（政策/天气/地缘/供需等）。

在实现时建议定义 Pydantic 或 TypedDict 模型（如 `SupplyNode`、`PricePoint`、`NewsItem`），并放入 `core/models/commodity_models.py`，供 state 与工具共用。

#### 4.2.1 实体 Schema 示例（Pydantic/TypedDict）

```python
# 供给节点
SupplyNode: region, commodity_id, capacity_or_output, inventory, trade_flow, key_entities_ports, source_url, source_name, as_of_date

# 价格点
PricePoint: commodity_id, market_or_contract, price_type(spot/future/index/spread), value, unit, as_of_date, source_url, source_name

# 新闻项
NewsItem: title, summary, source_name, source_url, published_at, related_commodities, related_regions, tags[](policy/weather/geopolitics/supply_demand)

# 原始证据包
RawEvidence: supply_chain: List[SupplyNode], prices: List[PricePoint], news: List[NewsItem], fetched_at, commodity_or_chain, time_range
```

### 4.3 数据流与存储

- 采集结果 → 先整理为「原始证据包」`RawEvidence`（含 supply_chain, prices, news 等列表）。
- 可落库：SQLite/JSON 文件/现有 `data/` 目录，带品种+时间维度，便于去重与增量。
- 缓存：对同一品种+时间范围的请求可返回缓存，并可选 TTL。

### 4.4 数据清洗与去重规则

- **去重**：供给/价格按「品种+来源+时间」去重；新闻按「标题+来源」或 URL 去重。  
- **归一化**：品种名、单位、日期格式统一为内部枚举或标准格式（如 ISO 日期、统一单位）。  
- **缺失与异常**：标记明显缺失字段；价格/数量异常值（如负值、偏离均值过大）可打标或剔除并记入日志。

### 4.5 存储与缓存策略

- **目录与键**：建议 `data/commodity/{commodity_id}/{date}.json` 存 RawEvidence；缓存键可为 `commodity_analysis:{commodity_or_chain}:{time_range_hash}`，TTL 如 1 小时。  
- **增量**：按品种+日期拉取时，可只拉取缺失日期区间，再与本地已有合并后写入。  
- **容量**：单品种单日 RawEvidence 控制在合理大小（如 < 500KB）；超长新闻列表可截断或分页存储。

## 七、Agent 与工作流设计

### 7.1 状态定义（CommodityAnalysisState）

在 `core/models/commodity_state.py` 中定义 TypedDict，建议包含：

- **输入**：`commodity_or_chain`（品种或产业链名）、`time_range`（可选）、`user_question`（可选）、`strategy_type`（可选，趋势/套利/套保/事件驱动）。
- **采集结果**：`raw_evidence`（RawEvidence）、`fetch_error`（可选）。
- **分析结果**：`structured_analysis`（摘要、分类、驱动、关联品种）、`judgment`（缺口、矛盾、建议搜索列表）。
- **自动搜索**：`search_queries`（本轮要执行的查询）、`search_results`（本轮结果）、`round_index`（当前轮次，用于限轮）。
- **技术指标**：`technical_indicators`（各类技术指标值）、`market_state`（市场状态：趋势/震荡/转折）。
- **策略生成**：`strategy_signals`（生成的策略信号列表）、`strategy_confidence`（策略置信度）。
- **回测评估**：`backtest_results`（回测结果）、`risk_metrics`（风险指标）。
- **最终输出**：`final_report`、`structured_output`（可给前端/API 的 JSON）。

### 7.2 节点与职责


| 节点名（建议）                              | 职责                                                                                      |
| ------------------------------------ | --------------------------------------------------------------------------------------- |
| `commodity_intent_node`              | 解析品种/产业链/时间/策略类型，标准化为内部品种 ID 或关键词。                                                           |
| `commodity_fetch_node`               | 调用 tools 从公开网站拉取供给、价格、新闻，组装 RawEvidence，写入 state。                                       |
| `commodity_structured_analysis_node` | 用 LLM 对 RawEvidence 做结构化分析（摘要、分类、驱动、置信度），写入 `structured_analysis`。                      |
| `commodity_judgment_node`            | 用 LLM 做自动判定：缺什么、矛盾点、关键驱动；输出「建议搜索查询」列表，写入 `judgment`。                                    |
| `commodity_search_node`              | 根据 `judgment.search_queries` 执行搜索（调用搜索工具/爬虫），结果合并进 `raw_evidence` 或单独 `search_results`。 |
| `technical_indicators_node`          | 计算各类技术指标（MA、MACD、RSI、ATR、布林带等），识别市场状态，写入 `technical_indicators` 和 `market_state`。       |
| `strategy_generation_node`           | 综合基本面与技术面分析，生成买卖策略信号，写入 `strategy_signals`。                                                   |
| `strategy_risk_node`                 | 评估策略风险，计算风险指标，写入 `risk_metrics`。                                                                     |
| `strategy_backtest_node`             | 对策略进行历史回测，验证有效性，写入 `backtest_results`。                                                              |
| `commodity_loop_decision`            | 条件边：若本轮生成了新搜索且未超轮次 → 再走一遍 structured_analysis + judgment；否则 → 策略生成。                     |
| `commodity_report_node`              | 根据 `structured_analysis` + `strategy_signals` + `backtest_results` 生成最终报告与 `structured_output`。               |


### 7.3 自动判定与自动搜索的闭环

- **自动判定（Judgment）**：LLM 输出结构化字段，例如：  
  - `missing_data`: 缺哪类供给/价格/新闻；  
  - `contradictions`: 多源冲突描述；  
  - `key_drivers`: 当前阶段最需关注的因子；  
  - `suggested_queries`: 用于下一步搜索的查询列表（中英文均可）。
- **自动搜索**：  
  - 仅执行 `suggested_queries` 中未在本轮执行过的查询；  
  - 搜索工具返回标题/摘要/链接（必要时可抓正文摘要）；  
  - 结果写入 state，并可选地 append 到 `raw_evidence.news` 或单独列表，供下一轮分析。
- **防循环**：  
  - 最大轮次（如 2～3 轮）；  
  - 或最大新增条目数（如单轮新增新闻 > 50 条则不再继续）；  
  - 或总耗时上限（如单次请求 > 5 分钟则进入报告生成）。满足任一即退出循环，进入策略生成。
- **条件边逻辑**：`commodity_loop_decision` 读取 `judgment.suggested_queries` 与 `round_index`；若 `suggested_queries` 非空且 `round_index < max_rounds` 则走「搜索 → 再分析」分支，否则走「策略生成」分支。

### 7.4 策略生成流程

- **技术指标计算**：  
  - 调用 `technical_indicators_node` 计算各类技术指标；  
  - 识别市场状态（趋势/震荡/转折）；  
  - 将结果写入 `technical_indicators` 和 `market_state`。
- **策略生成**：  
  - `strategy_generation_node` 综合基本面分析结果和技术指标；  
  - 根据用户指定的策略类型（或自动选择）生成策略信号；  
  - 确定入场点位、止损止盈、仓位规模；  
  - 评估风险收益比与策略置信度；  
  - 将结果写入 `strategy_signals`。
- **风险评估**：  
  - `strategy_risk_node` 计算策略风险指标；  
  - 进行压力测试；  
  - 将结果写入 `risk_metrics`。
- **策略回测**：  
  - `strategy_backtest_node` 对策略进行历史回测；  
  - 计算回测指标（收益率、夏普比率、最大回撤等）；  
  - 将结果写入 `backtest_results`。

### 7.5 工作流图（LangGraph）

- 入口 → `commodity_intent_node` → `commodity_fetch_node` → `commodity_structured_analysis_node` → `commodity_judgment_node` → 条件边 `commodity_loop_decision`：  
  - 若需继续 → `commodity_search_node` → 回到 `commodity_structured_analysis_node`；  
  - 否则 → `technical_indicators_node` → `strategy_generation_node` → `strategy_risk_node` → `strategy_backtest_node` → `commodity_report_node` → END。
- 所有 LLM 调用统一走 `core/tools/llm_client.py`，便于控制超时与错误。

### 7.6 Prompt 设计要点

- **结构化分析**：System 中明确输出格式（如 JSON 或 Markdown 小节），包含摘要、分类、驱动因子、关联品种、置信度；User 中注入 RawEvidence 的摘要或全文（按 token 限制截断）。  
- **自动判定**：要求 LLM 输出结构化字段：`missing_data`、`contradictions`、`key_drivers`、`suggested_queries`（列表，中英文均可）；并约定「无缺口时 suggested_queries 为空」，以便条件边判断是否进入搜索。  
- **策略生成**：要求 LLM 综合基本面与技术面，输出策略信号，包括策略类型、方向、入场点位、止损止盈、仓位规模、风险收益比、置信度等。  
- **风险评估**：要求 LLM 评估策略风险，输出 VaR、最大回撤、夏普比率等风险指标。  
- **报告生成**：基于最新 `structured_analysis`、`strategy_signals`、`backtest_results` 生成最终报告，包含供给链摘要、价格摘要、驱动因子、风险、建议、买卖策略；可同时输出一份 `structured_output`（JSON）供前端/API 使用。

### 7.7 错误处理与降级

- **采集失败**：单源超时或异常不阻塞整体；将错误信息写入 `fetch_error`，分析节点可基于部分证据继续；报告中可注明「某类数据缺失」。  
- **LLM 失败**：重试 1～2 次；若仍失败则跳过该节点，用默认或上一轮结果填充，并记录日志。  
- **搜索失败**：单条查询失败不阻塞；已成功结果合并进 state，未成功查询可记入 judgment 供人工或下次重跑。
- **策略生成失败**：若 LLM 无法生成有效策略，可返回空策略列表，报告中注明「当前市场条件下暂无明确交易机会」。
- **回测失败**：若历史数据不足或回测异常，可跳过回测步骤，报告中注明「回测数据不足」。

  - 在 `core/tools/commodity_search.py` 实现：根据查询列表调用公开搜索 API 或指定站点，返回标题/摘要/链接（及可选正文摘要）。  
  - 与 Agent 约定好输入/输出格式（如 List[SearchResult]），便于 `commodity_search_agent` 写回 state；支持中英文查询。  
  - 限频与超时；失败返回空列表并打日志。
4. **技术指标计算工具**
  - 在 `core/tools/technical_indicators.py` 实现：计算各类技术指标（MA、EMA、MACD、RSI、ATR、布林带、OBV等）。  
  - 实现市场状态识别（趋势/震荡/转折）。  
  - 输出统一格式，供策略生成 Agent 使用。
5. **回测引擎**
  - 在 `core/tools/backtest_engine.py` 实现：历史数据回测框架。  
  - 支持多种策略类型（趋势、套利、套保、事件驱动）。  
  - 计算回测指标（收益率、夏普比率、最大回撤、胜率、盈亏比等）。  
  - 考虑交易成本（手续费、滑点）。
6. **Agent 节点**
  - 实现 `commodity_intent_agent`：解析品种/产业链/时间/策略类型，标准化为内部品种 ID 或关键词，写入 state。  
  - 实现 `commodity_fetch_agent`：调 commodity_fetcher，将结果写入 `raw_evidence`。  
  - 实现 `commodity_analysis_agent`（结构化分析）+ `commodity_judgment_agent`（自动判定）：LLM 输出包含摘要、驱动、`suggested_queries` 等；输出格式可用 JSON schema 约束。  
  - 实现 `commodity_search_agent`：读 `judgment.search_queries`，调搜索工具，将结果合并进 `raw_evidence` 或 `search_results`，并可选去重。  
  - 实现 `strategy_generation_agent`：综合基本面与技术面，生成买卖策略信号，写入 `strategy_signals`。  
  - 实现 `strategy_risk_agent`：评估策略风险，计算风险指标，写入 `risk_metrics`。  
  - 实现 `strategy_backtest_agent`：对策略进行历史回测，验证有效性，写入 `backtest_results`。  
  - 实现 `commodity_report_agent`：根据 `structured_analysis`、`strategy_signals`、`backtest_results` 生成最终报告与 `structured_output`（JSON）。

7. **工作流图**
  - 在 `core/graph/commodity_analysis_graph.py` 用 LangGraph 组网：  
    - 入口 → intent → fetch → structured_analysis → judgment → 条件边（是否有 suggested_queries 且 round < max_rounds）：是 → search → 回到 structured_analysis（round_index+1）；否 → technical_indicators → strategy_generation → strategy_risk → strategy_backtest → report → END。
  - 编译为 `compiled_commodity_graph`；支持 `invoke(initial_state)` 或 `stream`。  
  - 配置项：max_rounds（如 2）、单次请求超时、LLM temperature。
8. **API**
  - 在 `server/routers/commodity.py` 提供 `POST /api/commodity/analyze`（body：commodity_or_chain、time_range、user_question、strategy_type）。  
  - 提供 `POST /api/commodity/strategy`（body：commodity、strategy_type、parameters）用于单独生成策略。  
  - 提供 `POST /api/commodity/backtest`（body：strategy_id、start_date、end_date）用于策略回测。  
  - 在 `server/app.py` 中 `include_router(commodity.router)`。  
  - 请求中构建初始 state，执行 `compiled_commodity_graph.invoke(...)`；返回 `report`、`structured`、`strategies`、`backtest_results`、可选 `rounds_used`；异常时返回 4xx/5xx 与 message。  
  - 可选：增加 GET 缓存查询、限流中间件。
9. **前端（可选）**
  - 新增「大宗商品分析」页：输入品种/产业链、时间范围、可选问题、策略类型；调用 API；展示供给链摘要、价格摘要、驱动因子、风险、建议、买卖策略、回测结果；支持加载态与错误提示。  
  - **展示结构建议**：顶部为查询表单；下方为报告正文（Markdown 或富文本）+ 结构化卡片（供给链摘要、价格摘要、驱动因子、风险、建议、买卖策略）；若有 `cached` 标记可显示「结果来自缓存」；长耗时请求可考虑 SSE 或轮询进度（若后端支持）。  
  - 新增「策略回测」页：选择策略、设置回测参数、展示回测结果图表（净值曲线、回撤曲线、收益分布等）。
10. **合规与鲁棒性**
  - 采集与搜索：限频、User-Agent、错误重试、超时；遵守各站 robots.txt 与条款。  
  - 敏感信息：不把 API Key 写进代码，使用环境变量。  
  - 日志：记录每轮判定与搜索查询、LLM 调用耗时、策略生成过程、回测结果，便于审计与调试。

## 九、API 设计

### 9.1 请求与响应

#### 9.1.1 综合分析端点

- **端点**：`POST /api/commodity/analyze`（或 `/api/commodity/analysis`）。  
- **请求体**：  
  - `commodity_or_chain`（string，必填）：品种或产业链名称，如「原油」「铜」「大豆」或「油脂油料」。  
  - `time_range`（object，可选）：`{ "start": "YYYY-MM-DD", "end": "YYYY-MM-DD" }`，不传则默认近期（如近 30 天）。  
  - `user_question`（string，可选）：用户追问或关注点，如「关注 OPEC 会议对供给的影响」。  
  - `strategy_type`（string，可选）：策略类型，如「trend」「arbitrage」「hedge」「event_driven」，不传则自动选择。  
  - `enable_backtest`（boolean，可选）：是否启用回测，默认 true。  
- **响应体**：  
  - `success`（boolean）、`message`（string，可选）、`data`（object）：  
    - `report`（string）：最终报告正文。  
    - `structured`（object）：供给链摘要、价格摘要、驱动因子、风险、建议关注点等结构化字段，便于前端展示。  
    - `strategies`（array）：生成的买卖策略信号列表。  
    - `backtest_results`（array，可选）：策略回测结果。  
    - `rounds_used`（int，可选）：实际执行的分析/搜索轮次。  
  - 错误时：`success: false`，`message` 或 `detail` 描述原因；HTTP 状态码 4xx/5xx。

#### 9.1.2 策略生成端点

- **端点**：`POST /api/commodity/strategy`。  
- **请求体**：  
  - `commodity`（string，必填）：品种名称。  
  - `strategy_type`（string，必填）：策略类型（trend/arbitrage/hedge/event_driven）。  
  - `parameters`（object，可选）：策略参数，如 `{ "ma_short": 5, "ma_long": 20, "risk_ratio": 0.02 }`。  
  - `time_range`（object，可选）：分析时间范围。  
- **响应体**：  
  - `success`（boolean）、`message`（string，可选）、`data`（object）：  
    - `strategies`（array）：生成的策略信号列表。  
    - `technical_indicators`（object）：技术指标值。  
    - `market_state`（string）：市场状态。

#### 9.1.3 策略回测端点

- **端点**：`POST /api/commodity/backtest`。  
- **请求体**：  
  - `strategy_id`（string，必填）：策略ID（或策略参数）。  
  - `start_date`（string，必填）：回测开始日期。  
  - `end_date`（string，必填）：回测结束日期。  
  - `initial_capital`（float，可选）：初始资金，默认 1000000。  
  - `commission`（float，可选）：手续费率，默认 0.0001。  
- **响应体**：  
  - `success`（boolean）、`message`（string，可选）、`data`（object）：  
    - `backtest_result`（object）：回测结果，包含收益率、夏普比率、最大回撤、胜率等指标。  
    - `equity_curve`（array）：净值曲线。  
    - `drawdown_curve`（array）：回撤曲线。  
    - `trade_list`（array）：交易记录列表。

#### 9.1.4 策略列表端点

- **端点**：`GET /api/commodity/strategies`。  
- **查询参数**：  
  - `commodity`（string，可选）：品种筛选。  
  - `strategy_type`（string，可选）：策略类型筛选。  
  - `limit`（int，可选）：返回数量限制，默认 20。  
- **响应体**：  
  - `success`（boolean）、`data`（object）：  
    - `strategies`（array）：策略列表。  
    - `total`（int）：总数。

### 9.2 错误码约定（示例）

- `400`：参数错误（如 commodity_or_chain 为空）。  
- `429`：超过限流。  
- `504`：分析超时（如 LLM 或采集超时）。  
- `500`：服务端异常（采集大面积失败、图执行异常等）；`detail` 中可包含简短原因，避免泄露内部路径。  
- `503`：回测数据不足或回测服务不可用。

### 9.3 限流与缓存

- **限流**：按 IP 或用户维度限制 QPS/分钟，避免滥用采集与 LLM。  
- **缓存**：对相同 `commodity_or_chain` + `time_range` + `strategy_type` 的请求，若存在未过期缓存（如 TTL 1 小时），可直接返回缓存结果并标记 `cached: true`。  
- **策略缓存**：对生成的策略信号可缓存较短时间（如 5-10 分钟），避免重复计算。

## 十一、安全与合规

- **爬虫规范**：遵守各站 robots.txt；请求间隔与并发可配置，避免对单站造成压力；User-Agent 标识为项目名/用途，便于对方识别。

- **密钥与配置**：API Key、数据库连接等使用环境变量或配置中心，不写死代码；日志中不打印敏感参数。  
- **审计**：关键操作（如每轮判定、搜索查询、报告生成）打日志，便于审计与排查。

## 十四、附录

### 14.1 品种/产业链示例（可配置扩展）

- **品种**：原油、布伦特、WTI、铜、铝、大豆、豆粕、豆油、铁矿、焦煤、焦炭、螺纹钢、黄金、白银等。  
- **产业链**：油脂油料（大豆→豆粕/豆油）、黑色（铁矿/焦煤→螺纹/热卷）、能源化工（原油→PTA/甲醇）等。

### 14.2 策略类型示例

- **趋势跟踪策略**：基于移动平均线、MACD、RSI等技术指标识别趋势方向，在趋势形成时入场，趋势反转时出场。  
- **跨期套利策略**：同一品种不同交割月份合约之间的价差交易，基于季节性规律、库存周期、供需预期。  
- **跨品种套利策略**：相关品种之间的价差交易，如原油与成品油、大豆与豆粕豆油、铁矿与螺纹钢。  
- **套期保值策略**：生产者卖出期货锁定销售价格，消费者买入期货锁定采购成本。  
- **事件驱动策略**：基于政策事件（OPEC+减产、美联储政策）、天气事件（干旱、洪涝）、地缘政治（战争、制裁）等驱动因素的交易策略。

### 14.3 技术指标示例

- **移动平均线（MA）**：简单移动平均（SMA）、指数移动平均（EMA），用于识别趋势方向和支撑阻力位。  
- **MACD**：指数平滑异同移动平均线，由DIF、DEA、MACD柱组成，用于判断趋势强度和转折点。  
- **RSI**：相对强弱指标，用于判断超买超卖状态，RSI>70为超买，RSI<30为超卖。  
- **布林带**：由中轨（SMA）、上轨、下轨组成，用于判断价格波动范围和突破信号。  
- **ATR**：平均真实波幅，用于衡量市场波动性，设置动态止损位。

### 14.4 风险指标示例

- **VaR（Value at Risk）**：在给定置信水平下的最大可能损失，如95%置信度下的VaR表示有5%概率损失超过该值。  
- **最大回撤**：从峰值到谷底的最大跌幅，衡量策略的最大风险。  
- **夏普比率**：风险调整后收益，等于（收益率-无风险利率）/波动率，衡量单位风险的超额收益。  
- **索提诺比率**：仅考虑下行风险的夏普比率变种，更关注下行风险。  
- **卡玛比率**：收益率/最大回撤，衡量单位最大回撤的收益。

### 14.5 术语表

- **RawEvidence**：原始证据包，包含供给、价格、新闻等列表。  
- **StructuredAnalysis**：LLM 产出的结构化分析（摘要、分类、驱动、置信度）。  
- **Judgment**：自动判定结果（缺口、矛盾、关键驱动、建议搜索查询）。  
- **TTL**：缓存存活时间（Time To Live）。  
- **StrategySignal**：策略信号，包含策略类型、方向、入场点位、止损止盈、仓位规模等。  
- **BacktestResult**：回测结果，包含收益率、夏普比率、最大回撤、胜率等指标。  
- **VaR**：风险价值，在给定置信水平下的最大可能损失。  
- **MA**：移动平均线。  
- **MACD**：指数平滑异同移动平均线。  
- **RSI**：相对强弱指标。  
- **ATR**：平均真实波幅。
